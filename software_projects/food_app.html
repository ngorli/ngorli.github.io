Recipe Sharing Social Media App

For my group senior project for my Computer Science degree, myself and three other group members created NENA: a social media app for sharing recipes. I will cover the portions of the iOS app that I worked on.

  • Overview
    • This mobile app is an iOS application made using firebase and React Native. The motivation for NENA was to allow people to have a creative outlet to express themselves in a culinary setting. They can share and upload recipes, view others peoples recipes, search for recipes, and add friends on the application.
  • Backend
    • The first part of the work I did on this application was my work on the backend of the project in Firebase. This consiisted of designing tje table heiracrhy in the backend, which was made up of recipes and users. Each user collection had fields for their DOB, cooking_level - where they could select from a list of levels - dietary restrictions, email address, first and last name, friends, friend requests, pending requests, passwords, user ids, and their profile picture url. The recipes collection contained lists for ingredients and instructions, whether or not the recipe is public, a recipe description, a recipe picture, a recipe title, a creation timestamp, and the user id.
    • When the backend architecture had been determined, my main task was to work on writing the backend functions that would be called on the front end and write information to the database in Firebase. The functions I worked on were functions to handle all of the processes in relating to friends in the application and processes used for creating a user and logging in.
  • Frontend
    • I also did work on the frontend of the application. Below are all the screens I worked on in this application
      • Home Screen The home screen is the landing page when users log in. I designed the layout of this screen as well as making it possible for users to see the number of friends and recipes they have, as well as images for their recipes. Clicking on a recipe on the home page would also redirect to that recipe and clicking on the numner of friends would redirect to the friends page.
  • Friend Screen The friend screen lists the names of all of the users friends. It also allows users to type in the name of a different user to send a freind request to as well as accept or deny a friend request. I worked on the layout and functionality of this screen.
  • Feed Screen The feed screen allows users to scroll through all of the recipes that their friends have uploaded. The recipe titile, recipe description, user, and recipe image are all displayed. I worked on the layout and functionality of this screen.
  • Search Screen The search screen allows users to search for any recipes that they are interested in. The recipes would come up with with the recipe titile, recipe description, user, and recipe image. From this page, users can click on the recipe and bring up the recipe landing page. I worked on the layout and functionality of this screen.
  • Recipe Screen The recipe screen is where users can see a given recipe. This screen includes ingredients, a picture, and step by step instructions for how to create the recipe.
